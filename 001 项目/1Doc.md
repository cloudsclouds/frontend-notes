# 协同文档

1. 撤销重做为什么不用全量快照？

这是高频问题。
因为全量快照虽然实现简单，但问题也很明显：

- 内存占用大
- 高频编辑时快照膨胀
- 序列化成本高
- 长链路性能差
于是很多项目会引入：
- diff 增量记录
- 命令模式
- 操作日志流

三种方案分别适用于什么场景？  


|         |            |           |       |
| ------- | ---------- | --------- | ----- |
| 方案      | 优点         | 缺点        | 场景    |
| 全量快照    | 简单稳定       | 内存占用高     | 小型编辑器 |
| diff 增量 | 节省空间       | diff 计算复杂 | 文本场景  |
| 命令模式    | 易扩展、适合复杂交互 | 设计复杂      | 低代码平台 |

## SSE 流式输出
### 1. 为什么用 SSE 不用 WebSocket？
AI 对话场景本质上更多是“服务端单向推流”。
所以 SSE 会天然更轻量：
- 基于 HTTP
- 自动断线重连
- 部署简单
- 更适合文本流式生成

### 2. SSE 有什么缺点？
- 只能服务端单向通信
- 浏览器连接数限制
- 长连接稳定性问题
- 不适合高频双向实时通信
