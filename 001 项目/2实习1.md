# 封装组件
## 封装 FollowTable 组件
1. 封装组件的背景是这样的，原生的 sticky 定位在处理嵌套容器滚动的时候，会出现吸顶式小的问题。
为了保证在复杂场景下的稳定性，放弃了纯 CSS 方案，采用 JavaScript 监听滚动事件，通过选择滚动容器、计算元素位置、复制表头来实现吸顶。
虽然成本稍高一些，但能够解决原生方案在复杂场景下的稳定性问题。

2. 另一个问题是，在做多表格页面的时候，发现当用户对最后一个表格进行筛选，数据变少时，页面总高度会突然收缩，导致底部内容跳动，体验很差。
为了解决这个问题，我在组件初始化时，专门计算了最后一个表格到底部的距离，并将其锁定。无论表格数据如何变化，都强制保持着底部间距不变，从而实现了视觉上的防抖动。

# 首屏优化
针对首屏 FCP 和 LCP 指标较差的瓶颈，我设计了分阶段渲染策略。首先，将页面拆分为关键路径和非关键路径，首屏之加载核心的纵览信息，而将复杂的详情表格通过 defineAsyncComponet 进行按需加载，避免阻塞主要线程。同时，还利用浏览器空闲时间，对后续 Tab 资源进行预加载。
最终显著降低了首屏耗时，提升了用户体验。 
FCP 和 LCP 通过浏览器自带的 LightHouse 面板看。
